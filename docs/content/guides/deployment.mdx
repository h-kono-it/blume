# Deployment

## Static

The default. `blume build` writes `dist/` — deploy it to any static host.

```bash
blume build
```

Static builds include local search, `sitemap.xml` (when `deployment.site` is
set), redirects, and `llms.txt`.

## Server

For dynamic features, switch to server output and pick an adapter:

```ts
deployment: {
  output: "server",
  adapter: "vercel", // vercel | node | netlify | cloudflare
}
```

## Redirects

```ts
redirects: [{ from: "/old", to: "/new", status: 301 }];
```

Static builds emit redirect pages; server builds handle them at request time.

## Open Graph images

Enable rendered OG images (1200×630) for every page, generated at build with
[Takumi](https://takumi.kane.tw) — no headless browser:

```ts
og: {
  enabled: true;
}
```

Blume prerenders a PNG per route at `/og/<slug>.png` and adds `og:image` and
Twitter card meta tags. Set `deployment.site` for absolute URLs, and override a
single page with `seo.image` in frontmatter.

## Diagnostics

Every build prints a summary of the output mode, adapter, search provider,
redirect count, sitemap and llms status, and any enabled server features.
